	2.	컴퓨터로 업무를 진행한다면?

		나 	: 	한국어 사용.

		컴퓨터 	:  	기계어 사용

				기계어 - 컴퓨터가 이해할 수 있는 0과 1로 구성된 언어체계.



	3.	사람과 컴퓨터사이의 공통의 대화수단이 필요하다.

		-	하지만 사람이 기계어를 공부한다거나 컴퓨터에게 사람의 언어를 가르친다는 것은 

			아직까지는 불가능.


	4.	프로그래밍 언어와 컴파일러의 등장.
		사람의 언어에 가까운 명령체계인 프로그래밍 언어와 
		이를 기계어로 번역해주는 컴파일러가 등장하였다.
		
		나			-	컴파일러		-	컴퓨터
	    	( 한국어, C언어 )	 	( C언어, 기계어 )         	( 기계어 )

2.	프로젝트 생성하기
---------------------------------
	1.	파일 	- 새로 만들기 		- 프로젝트 	- 빈 프로젝트 선택

			- 솔루션 탐색기 우클릭	- 새 항목 	- ***.c 생성.

		* 	*.c 	: 	c 문법을 준수하는 프로그래밍.
			*.cpp	:	c++ 문법을 준수하는 프로그래밍.
	

---------------------------------
3.	빌드 과정
---------------------------------
	1.	전처리 과정( 프리 프로세싱 )
		:	#include, #define 처럼 "#"이 들어간 문법들을 코드로 변환.

	2.	컴파일( by 컴파일러 )
		:	전처리 과정에서 생성된 파일들로부터 문법적 오류를 검사하여
			오류가 없으면 어셈블리 코드로 번역.

			* 어셈블리 코드
			:	기계어와 1:1 대응되는 저급 언어

			* 저급언어
			:	기계어에 가까운 언어로 사람이 가독하기엔 힘들지만 속도는 빠르다.

	3.	어셈블링( by 어셈블러 )
		:	어셈블리 코드를 obj 파일로 변환.
		
	4.	링킹
		:	obj 파일을 연결하여 실행파일 생성.
