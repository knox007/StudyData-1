/// IPC ////////////////////////////////////////////////////////////////////
------------
1.	IPC란?
------------
	1.	각 프로세스들은 메모리와 CPU를 공유할 뿐 서로 독립된 일을 하며 상호 간섭하지 않고
		간섭할 수도 없다.
	
	2.	필요에 따라 프로세스간에 정보를 교환해야할 경우가 생기는데 이때 정보를 교환하는 방법을
		IPC( InterProcess Communication )이라고 함.
	
	3.	메시지, 클립보드, 파일 맵핑, 윈속, 원격함수 호출(RPC) 등의 방법이 있음.


	4-1.	메시지 방식
			1.	가장 간단하고 빠른 방법.
			2.	두 프로세스가 서로의 윈도우 핸들만 알고 있다면 wParam과 lParam을 통해 정보 교환 가능.
			3.	메모리를 거치지 ㅇ낳고 운영체제에 의해 직접 전달.
			4.	비교적 크기가 작은 정보전달에 사용.
				-	문자열이나 크기가 큰 구조체는 전달 불가능.
				-	wParam, lParam은 둘다 4바이트에 불과하다.
		
			5.	"1.IPCMessage" 참고.

	4-2.	IPC 전용 메시지 방식.
			
			1.	WM_COPYDATA 메시지 활용.
			
			2.	wParam	-	대상 윈도우의 핸들.
				lParam	-	COPYDATASTRUCT의 포인터.

				struct tagCOPYDATASTRUCT
				{
					ULONG_PTR dwData;	//	교환하고자 하는 정수값.
					DWORD     cbData;	//	lpData의 크기
					PVOID     lpData;	//	교환하고자 하는 문자열(또는 이진 데이터)
										//	포인터이므로 구조체나 배열도 가능.
				} COPYDATASTRUCT
	
			3.	주의 사항
				1.	통신하는 양쪽이 데이터의 포맷을 알고 있어야 함.
				2.	받은 쪽에서 데이터를 다 사용하기 전에 전달된 값이 바뀌지 않도록 해야함.
					멀티스레드를 쓸 경우 다른 스레드에서 값을 변경할 여지를 차단해야함.
