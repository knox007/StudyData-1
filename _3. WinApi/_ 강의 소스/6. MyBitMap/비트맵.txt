/// 비트맵 ////////////////////////////////////////////////////

1. 정의	:	디지털이미지를 저장할때 가장 기본적으로 쓰이는 포멧.

			화소라는 기본 단위를 픽셀로 나타냄.

			1픽셀 마다 고유의 색상을 담고 있다.

			무수히 많은 픽셀들이 한개의 이미지를 나타냄.

			가로 X 세로 만큼의 픽셀정보를 모두 저장해야하므로 다른 이미지 포맷에 비해 용량이 크다.
			->	이를 개선하기 위해 다양한 압축포맷들이 개발 되었다.( JPEG, GIF, PNG 등.. )

			(비트맵 정보.txt / 비트맵 분석 폴더 참고)


2.	BOOL  BitBlt(	HDC hdcDest, 
					int destX, int destY, int width, int height,
					HDC hdcSrc,
					int srcX, int srcY, DWORD rop)
	
	-	기능 : DC간의 영역끼리 고속 복사 수행.
	
	-	hdcDest : 복사 대상 DC.
	-	destX, destY, width, height : 복사대상의 위치, 크기.
	-	hdcSrc : 복사원 DC.
	-	srcX, srcY : 복사원 좌표.
	-	rop :	래스터 연산 방법.
				SRCCOPY, BLACKNESS, WHITENESS, DSTINVERT 등이 있음.


3.	BOOL  StretchBlt(	HDC hdcDest, 
						int xDest, int yDest, int wDest, int hDest,
						HDC hdcSrc, 
						int xSrc, int ySrc, int wSrc, int hSrc,
						DWORD rop);
	
	-	기능 : BitBlt 유사하나 확대, 축소가 가능.

	-	wSrc, hSrc : 복사원의 크기.
	-	wDest / hDest 가 wSrc / hDest 보다 크거나 작으면
		확대 또는 축소가 된다.


** 이런 복잡한 그래픽 처리까지 CPU가 전담했으나
   Direct X나 OPENGL등의 그래픽 랜더러 프로그램이 등장하여
   GPU에 넘김으로서 CPU의 부담이 확연히 줄어들었다.
